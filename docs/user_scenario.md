# 마이링크 사용자 시나리오 (User Scenario)

본 문서는 PRD 요구사항에 기반하여 작성된 마이링크(My-Link)의 핵심 사용자 흐름입니다.

## 1. 방문자 (Visitor) 시나리오

**마이링크 프로필 뷰어 페이지 방문 과정**
*   사용자는 **찾고자 하는 특정 사용자의 정보와 링크 목록들을 한눈에 확인**하기 위해 **제공된 고유 프로필 주소(`mylink.com/디스플레이네임(displayName)`)를 인터넷 브라우저에 입력하여 접속**한다.
*   사용자는 **작성자가 연결해 둔 소셜 미디어, 포트폴리오, 혹은 다른 목적지 웹사이트로 직접 이동**하기 위해 **화면에 렌더링 된 여러 개의 링크 아이템 중 원하는 것을 클릭**한다.

---

## 2. 소유자 (Owner) 시나리오

**가입 및 초기 설정 과정**
*   사용자는 **복잡한 이메일 인증 없이 편리하게 가입 및 로그인**을 하기 위해 **구글(Gmail) 계정을 연동 로그인 수단으로 선택하여 인증**한다.
*   사용자는 **자신만의 고유한 프로필 URL 슬러그(Slug)를 손쉽게 설정**하기 위해 **가입 직후 구글 지메일 주소의 아이디 부분(@ 앞 텍스트)이 '디스플레이 네임(displayName)'으로 추출 및 자동 세팅되어 기본 접속 주소(`mylink.com/디스플레이네임`)로 반영된 것을 확인**한다.

**관리자 대시보드 - 프로필 및 링크 관리 과정**

### 프로필 수정
*   사용자는 **디스플레이 네임이나 실제 자신의 유저네임(username)을 변경**하기 위해 **미리보기 화면 영역의 텍스트를 클릭(인라인 편집 활성화)하여 곧바로 수정**한다.

### 링크 추가
*   사용자는 **나의 프로필에 새로운 홍보용 주소를 추가로 노출**하기 위해 **대시보드 화면에 위치한 '새 링크 추가' 버튼을 클릭**한다.
*   사용자는 **이 링크가 무엇을 의미하는지 방문자가 직관적으로 알 수 있게** 하기 위해 **방금 생성된 빈 링크 항목의 '제목(타이틀)' 텍스트 영역을 클릭(인라인 편집)하여 문구를 키보드로 입력**한다.
*   사용자는 **방문자를 실제로 목적지로 연결**하기 위해 **'URL' 텍스트 영역을 클릭(인라인 편집)하여 대상 웹사이트의 전체 주소를 복사 및 붙여넣기(또는 입력)**한다.
*   (자동 적용) 사용자가 위와 같이 URL 입력을 마치면, 시스템은 **방문자가 해당 링크 항목을 즉각적으로 알아볼 수 있도록** 하기 위해 **해당 URL 도메인을 바탕으로 구글 파비콘 API를 호출해 아이콘 이미지를 추출하고, 링크 항목 좌측에 자동으로 노출**시킨다.

### 링크 수정 및 삭제
*   사용자는 **기존에 등록해 둔 링크의 오타를 바로잡거나 주소를 변경**하기 위해 **수정하고자 하는 기존 링크 아이템의 텍스트(타이틀 혹은 URL)를 직접 클릭**한다.
*   사용자는 **새로운 값을 입력하고 변경 사항을 최종적으로 저장**하기 위해 **키보드의 'Enter' 키를 누르거나 화면의 빈 공간(다른 영역)을 클릭하여 입력 상태를 종료(Focus Out)**한다.
*   사용자는 **현재는 종료된 행사 링크 등 더 이상 유효하지 않은 특정 링크를 프로필에서 아예 제거**하기 위해 **해당 링크 아이템 측면에 위치한 '휴지통(삭제)' 아이콘을 클릭**한다.

---

## 3. 질문 및 개선 제안 사항 (Edge Case)

위의 시나리오를 바탕으로 개발을 진행하기 위해 기획 내용의 보완이 필요한 **모호한 부분 및 개선점** 제안 항목입니다.

**1. 불완전한 링크의 퍼블릭 뷰 노출 여부 (제안)**
*   **상황:** 제목과 URL 중 하나만 입력된 불완전한 링크 항목의 노출 정책이 없습니다.
*   **제안:** 제목과 URL 필드가 **모두 채워진('유효한') 상태일 때에만** 방문자가 보는 퍼블릭 뷰어에 렌더링 되도록 숨김 처리(Validation) 기능을 추가하는 것이 구조적으로 안전합니다.

**2. 링크 삭제 시 안전장치 부재 (질문 및 제안)**
*   **상황:** 한 번의 '휴지통' 클릭만으로 데이터가 삭제되는 동작에 대한 되돌리기 여부가 명시되어 있지 않습니다.
*   **제안:** 삭제 확인 모달 팝업 대신 유저 피로도가 낮은 인라인 개선 방식을 추천합니다. 휴지통 버튼 클릭 시 즉시 화면에서는 링크를 지우되, **화면 하단에 3초 동안 '삭제되었습니다. [실행 취소(Undo)]' 토스트(Snackbar) 기반의 복구 알림 옵션**을 제공하면 더 세련된 UX를 만들 수 있습니다.

**3. 존재하지 않는 혹은 탈퇴한 프로필 방문 시 (제안)**
*   **상황:** 방문자가 오타를 내거나 잘못된 주소(예: `mylink.com/없는아이디`)로 마이링크를 방문했을 때의 동작이 없습니다.
*   **제안:** 접속 시 "존재하지 않거나 삭제된 프로필입니다"라는 안내 페이지(404 Page)를 띄우고, 페이지 중앙에 **"나만의 마이링크 만들기" CTA(Call to Action) 버튼을 배치하여 자연스러운 회원가입을 유도**하는 마케팅 흐름을 추가하는 것이 좋습니다.
