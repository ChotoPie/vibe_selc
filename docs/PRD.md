# 마이링크 (My-Link) 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요

*   **프로젝트명:** 마이링크 (My-Link)
*   **목적:** 사용자가 자신의 여러 소셜 미디어, 웹사이트 등의 링크를 단일 페이지에 모아 공유할 수 있는 '멀티 링크 프로필' 서비스 제공 (Linktree 클론)
*   **대상 사용자:** 인플루언서, 크리에이터, 소상공인 등 여러 플랫폼에서 활동하며 하나의 통합된 프로필 링크가 필요한 사람

---

## 2. 핵심 기능 목록

*   **사용자 인증 및 계정 관리:** 구글 소셜 로그인 연동(또는 지메일 기반 가입/로그인), 로그아웃 처리
*   **고유 프로필 URL (Slug) 설정:** 사용자의 **닉네임(displayName)**을 기반으로 `mylink.com/닉네임` 형태의 프로필 접속 URL 할당
*   **프로필 관리 (인라인 편집):** 
    *   초기 가입 시 **구글 지메일(Gmail) 아이디의 앞부분(@ 앞 텍스트)**을 가져와 **디스플레이 네임(displayName)** 기본값으로 자동 설정
    *   사용자의 실제 이름을 보여주는 **유저네임(username)** 표시
    *   URL로 사용되는 **닉네임(displayName)** 과 **유저네임(username)**, 그리고 짧은 소개글을 **인라인 편집(Inline Editing)**으로 즉각 수정
*   **링크 관리 (인라인 편집):** 
    *   새로운 웹사이트 링크(URL + 타이틀) 추가
    *   등록된 링크의 텍스트와 대상 URL을 **인라인 편집**으로 즉시 수정 및 항목 삭제
    *   구글 파비콘 API(Google Favicon API)를 사용하여 각 링크 대상 사이트의 파비콘을 추출, 아이콘으로 매핑하여 표시
*   **프로필 뷰어 (퍼블릭 뷰):** 방문 접속자가 URL 검색 시 렌더링되어 보이는 최종 프로필 상세 및 링크 목록 웹 페이지

---

## 3. 기능 상세 설명

### 사용자 인증 및 계정 관리
*   사용자는 구글 계정(Gmail)을 기반으로 빠르고 간편하게 가입 및 로그인을 진행합니다.

### 프로필 관리
*   가입 초기, 사용자의 지메일 주소에서 아이디 부분(예: `abc@gmail.com` 의 경우 `abc`)을 추출하여 접속 URL의 기준이 되는 **닉네임(displayName)**으로 자동 등록합니다.
*   프로필 영역에는 사용자를 나타내는 **두 가지 이름 정보**가 노출됩니다.
    1.  **닉네임(displayName):** 고유한 접속 URL 슬러그(Slug) 식별자 역할이자 별명.
    2.  **유저네임(username):** 일반 방문자에게 신뢰감을 줄 수 있는 실제 사람 이름 혹은 브랜드명으로 사용되는 명칭.
*   닉네임, 유저네임, 소개글 텍스트를 클릭하면 즉시 입력 박스로 바뀌는 **인라인 편집**을 적용하여 별도의 화면 이동 또는 모달창 팝업 없이 그 자리에서 직관적으로 프로필 정보를 수정할 수 있습니다.
*   이미지 업로드 기능과 프로필 페이지 테마 커스텀(배경색, 버튼 등) 기능은 현재 제공하지 않습니다.

### 링크 관리 (대시보드)
*   사용자는 대시보드 안에서 자유롭게 새로운 링크를 추가하고 삭제할 수 있습니다.
*   링크의 '타이틀 텍스트'와 '목적지 URL' 영역 역시 클릭하여 즉시 입력 상태로 변환되는 **인라인 편집**을 통해 수정이 이루어집니다. (드래그 앤 드롭을 통한 목록 순서 변경이나 모바일 환경 미리보기 기능은 현재 없습니다)
*   새 링크가 입력되거나 URL이 변경될 때 즉시 구글 파비콘 API(`https://s2.googleusercontent.com/s2/favicons?domain={url}`)를 호출해 해당 주소의 실제 파비콘 아이콘을 아이템 왼측에 눈에 띄게 자동 적용합니다.

### 프로필 뷰어 (Public View)
*   방문자가 `[도메인]/displayName` 형식으로 브라우저에 접속했을 때 노출되는 실제 프로필 페이지입니다.
*   접속하는 기기(모바일, 랩탑, PC, 태블릿 등) 해상도에 자연스럽게 대응하는 반응형 웹으로 구현됩니다.

---

## 4. 데이터베이스 모델링 주요 사항

*   사용자(User) 문서에는 프로필 URL 식별자 역할인 `displayName`(필수 지정 필드)과 사용자 실제 이름인 `username` 필드가 개별로 존재해야 합니다.
*   사용자가 등록하는 링크 데이터는 배열 데이터 구조가 아닌, 반드시 사용자 문서 하위의 **서브 컬렉션(Sub-collection)**으로 분리하여 안전하게 저장하고 관리합니다.

---

## 5. 추후 추가 예정 기능 (이번 배포 제외)

*   **링크 클릭 통계:** 링크 각각이 클릭된 총 횟수와 같은 페이지 내부 조회수 지표를 확인할 수 있는 통계 기능은 향후 별도의 버전에 추가 업데이트될 예정입니다.
